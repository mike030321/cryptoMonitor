# Proposed engine design

## 1. Scope

Initial engine:

```json
{
  "engine": "sparse_post_cost_v1",
  "coin": "bitcoin",
  "timeframe": "5m",
  "mode": "paper_only",
  "executor_family": "volatility_defensive_or_sparse_opportunity_core",
  "other_families": "observe_only"
}
```

Only one execution profile should be allowed at first.

Do not let all 4 families trade the same signal.

If you think a new internal profile like `sparse_opportunity_core` is cleaner than reusing `volatility_defensive`, say so.

## 2. Label design

Replace the old `{UP, STABLE, DOWN}` objective with opportunity labels:

```text
LONG_OPPORTUNITY
SHORT_OPPORTUNITY
NO_TRADE
```

For BTC/5m, use:

```json
{
  "horizon_candles": 12,
  "horizon_minutes": 60,
  "round_trip_cost_pct": 0.30,
  "required_net_edge_pct": 0.45,
  "gross_move_required_pct": 0.75,
  "max_adverse_move_pct": 0.45
}
```

Definition:

* `LONG_OPPORTUNITY` only if future upside move over the horizon beats round-trip cost + required net edge, without violating max adverse excursion first.
* `SHORT_OPPORTUNITY` only if future downside move beats round-trip cost + required net edge, without violating max adverse excursion first.
* otherwise `NO_TRADE`.

If you disagree with these parameters, propose better ones based on actual BTC/5m distribution in the repo.

## 3. Model design

Use two heads:

### Classifier

Predicts:

```text
LONG_OPPORTUNITY / SHORT_OPPORTUNITY / NO_TRADE
```

### Regressor

Predicts:

```text
expected_net_return_after_costs
```

A trade is allowed only if both agree:

```text
classifier says opportunity
AND regressor expected_net_return_after_costs exceeds threshold
```

Do not rely on direction classifier alone.

## 4. Calibration design

Calibration should not be ignored, but should be interpreted correctly.

Rules:

* overconfidence blocks trading
* underconfidence blocks dynamic sizing but may allow fixed-size paper diagnostic
* dynamic sizing disabled until calibration is trustworthy

Suggested config:

```json
{
  "cal_dev_ceiling_dynamic_sizing": 0.20,
  "cal_dev_ceiling_fixed_size_paper": 0.55,
  "overconfidence_block": true,
  "underconfidence_fixed_size_allowed": true,
  "dynamic_sizing_allowed": false
}
```

Review this carefully.

If the model is underconfident but financially profitable, fixed-size paper may be allowed.
If it is overconfident, block.

## 5. Trade gate

Suggested initial BTC/5m gate:

```json
{
  "min_opportunity_prob": 0.65,
  "min_expected_net_return_pct": 0.45,
  "min_prob_gap": 0.20,
  "max_no_trade_prob": 0.40,
  "max_spread_bps": 8,
  "max_trade_rate": 0.025
}
```

If actual BTC/5m candidate evidence suggests different values, propose calculated alternatives.

## 6. Financial truth gate

For a candidate to enter diagnostic paper mode:

```json
{
  "min_forward_holdout_trades": 50,
  "min_net_pnl_pct": 0.0,
  "min_profit_factor": 1.20,
  "max_drawdown_pct": 8.0,
  "min_win_rate_pct": 50.0
}
```

These are safety gates for paper diagnostic, not dynamic live sizing.

## 7. Paper execution

For the first proof run:

```json
{
  "position_size_pct": 0.5,
  "max_open_positions": 1,
  "daily_loss_stop_pct": 1.5,
  "total_drawdown_stop_pct": 5.0,
  "review_after_hours": 72,
  "review_after_closed_trades": 50,
  "auto_disable_if_negative_after_closed_trades": 50
}
```

This is paper only.

No real wallet.
No global quant.

## 8. Meta-brain

Meta-brain remains shadow:

```json
{
  "mode": "shadow",
  "can_change_size": false,
  "can_suppress": false,
  "can_record_outcomes": true,
  "can_display_advice": true
}
```

Meta-brain may observe outcomes, but cannot control the engine yet.

## 9. Runtime behavior

The new engine must be isolated.

Required behavior:

* only BTC/5m can emit executable paper signals
* only the selected execution profile can place paper trades
* all other coins/timeframes remain blocked or observe-only
* 1h/2h/6h/1d remain context-only
* legacy agents remain archived
* no model that failed truth gate is discoverable by production registry
* research candidates stay quarantined until explicitly approved

## 10. Dashboard behavior

Dashboard must clearly show:

```text

Not global quant
Not real money
Meta-brain shadow only
Dynamic sizing disabled
```

Show:

* drawdown

* stopped / active status
* stop reason if disabled

Do not show this as the full quant brain being live.
