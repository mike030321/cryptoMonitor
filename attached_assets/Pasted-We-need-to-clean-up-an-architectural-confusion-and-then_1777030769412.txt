## First: correct the terminology

Do **not** treat these as the same thing.

### A. Quant Brain

This is the per-slice market predictor:

* per-(coin, timeframe) LightGBM specialists
* regime routing
* calibrated probabilities
* expected return
* live prediction path

This is the only component that predicts market movement directly. 

### B. Meta-model / stacking head

This is the learned combiner over slice outputs.

This is the component that may consume:

* per-slice calibrated probabilities
* per-slice confidence / recent accuracy
* pooled-fallback flags
* regime
* friction / fee / spread context
* cross-coin context

And output something like:

* buy / hold / abstain
* or sizing / confidence / abstain

This is what the recent proposal is actually describing.

### C. Meta-Brain

This is the already-shipped governance layer:

* trust by family
* defensive mode
* suppression / caution
* episodic / regime memory
* outcome-driven learning
* benchmark-aware governance

This is broader than a single LightGBM vote-combiner. It is already documented in the app architecture as a separate system under the vendored `market_meta_brain` package and its host wiring.
