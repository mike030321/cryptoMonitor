# Phase 2 — If calibration fails, prepare redesign plan

If final calibration repair fails to produce a clean paper candidate, write a redesign proposal for:

# Sparse Post-Cost Opportunity Engine

The proposed new engine should replace the broad directional engine with a rare-trade opportunity detector.

## New engine principle

Do not predict every candle.

Ask:

> “Is this one of the rare setups where a trade is worth taking after fees, slippage, spread, and risk?”

## Initial scope

Start tiny:

* BTC only
* 5m only
* paper only
* fixed-size only
* one execution profile only
* meta-brain shadow only
* no dynamic sizing
* no global quant enablement
* no other coins
* no other timeframes

## Proposed labels

Replace `{UP, STABLE, DOWN}` with:

* `LONG_OPPORTUNITY`
* `SHORT_OPPORTUNITY`
* `NO_TRADE`

Suggested starting parameters:

```json
{
  "timeframe": "5m",
  "horizon_candles": 12,
  "horizon_minutes": 60,
  "round_trip_cost_pct": 0.30,
  "required_net_edge_pct": 0.45,
  "gross_move_required_pct": 0.75,
  "max_adverse_move_pct": 0.45
}
```

If the repo’s BTC/5m distribution suggests better values, calculate and propose them.

## Proposed model

Use two heads:

1. classifier:

   * `LONG_OPPORTUNITY`
   * `SHORT_OPPORTUNITY`
   * `NO_TRADE`

2. regressor:

   * expected net return after costs

Trade only when both agree:

* classifier says opportunity
* regressor says expected net return exceeds threshold

## Proposed trade gate

Review or calculate better values, but start from:

```json
{
  "min_opportunity_prob": 0.65,
  "min_expected_net_return_pct": 0.45,
  "min_prob_gap": 0.20,
  "max_no_trade_prob": 0.40,
  "max_spread_bps": 8,
  "max_trade_rate": 0.025
}
```
